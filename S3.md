# S3 Replication
### SRR or CRR
* Same region replication or cross region replication
* Bucket versioning must be enabled for both source and destination buckets
* Replication is done asynchronously
* Only new objects will be replicated, Exisiting objects can be replicated using Batch Replication
* Delete markers can be replicated, but permanent delete are not
## Use cases
* SRR: log aggregation, replication across test and production 
* CRR: Compliance, low latency access, replication across accounts
* S3 GP: For  general purpose, low latency, big data analytics, gaming application, content distribution
* S3 IA: Disaster Recovery, backup, lower cost but there is a cost on retrieval
* S3 Glacier storage instant retrieval : Low cost archival tier, qucick access. Min duration 90 days
* S3 Glacier storage Flexible retrieval : 3 tiers of retrieval timing (1-5 min), (3-5 hrs), (5-12 hrs). Min duration 90 days
* S3 Glacier storage Deep archive : 2 tiers of retrieval timing (12 hrs), (24 hrs). Min duration 180 days
* S3 Intelligent tiering: Move objects between tiers
